HomeHive Codebase Report
Overview
HomeHive is a static frontend web application designed as a rental discovery platform. It allows users to search, filter, and view mock property listings (PGs, Rooms, Flats, Penthouses), read and submit reviews, and contact the platform owners.

The project is built using basic, modern web technologies: HTML5, Vanilla CSS, and Vanilla JavaScript (no external framework like React or styling libraries like TailwindCSS are used).

Project Structure
The repository consists of the following key files:

index.html (Landing Page)
reviews.html (Reviews & Ratings Page)
contact.html (Contact Us Page)
style.css (Global Stylesheet)
script.js (Core Application Logic)
OIP.jpg (Logo/Image Asset)
Detailed File Analysis
1. index.html (Home Page)
This is the main entry point to the application. It consists of the following sections:

Header: Contains the site logo and the main navigation bar linking to the three core pages.
Hero Section: A catchy headline stating "Find Your Perfect Stay, Instantly."
Search & Filter: A dedicated section allowing users to:
Search by text (location/property name).
Open a "Filter" dropdown menu filtering properties by:
Type (PG, Room, Flat, Penthouse)
Price Range
Food Availability (Yes/No)
Toggle between Dark Mode and Light Mode.
Listings: An empty container (<div id="listingsGrid">) which is populated dynamically by JavaScript.
Sidebar & Footer: Offers "Why HomeHive?" features and basic legal/contact links.
2. reviews.html
A page dedicated to property ratings.

Displays a hardcoded summary statistic (4.7/5.0 average based on 2450+ reviews) with visual progress bars.
Contains a Submit Review Form where users can select a star rating (powered by JS).
Displays a grid of mock user reviews styled as cards.
3. contact.html
Provides ways for users to get in touch.

Features a comprehensive Contact Form (Name, Email, Phone, Subject, Message) that is intercepted by JS upon submission.
Contains an Aside Section with hardcoded office address, phone, email, and business hours.
Integrates an embedded Google Maps iframe to show the office location.
Includes a short FAQ Section at the bottom.
4. script.js (Frontend Logic)
This file handles the interactivity of the website. Key functionalities include:

Mock Data: Defines rentalListings, an array of 8 Javascript objects representing accommodations.
Dynamic Rendering: renderListings() maps the mock data array into HTML string cards and injects them into #listingsGrid.
Filtering & Search: Contains applyFilters(), resetFilters(), and filterListings(). It listens to user input, updates a currentFilters state object, filters the rentalListings array, and re-renders the grid. Search input uses a custom debounce mechanism (debounce()) to prevent rendering lag while typing.
Form Submission: handleContactSubmit() intercepts the contact form submission, validates required fields and email formatting (regex), and simulates a success message without triggering a page reload.
Theme Toggling: toggleTheme() toggles a .dark-mode CSS class on the <html> root element. It stores the preference in the browser's localStorage so the theme persists across page reloads.
Interactive Stars (Reviews): setRating() and hover event listeners create a dynamic 5-star selection tool for the review form.
5. style.css (Styling)
A large, comprehensive stylesheet (over 1700 lines) handling global design.

Uses CSS Variables (:root) extensively to manage theme colors, spacing, and shadows for both light mode and .dark-mode.
The design exhibits premium UI patterns like Glassmorphism (backdrop-filter: blur), CSS gradients (linear-gradient), and box-shadow glowing effects.
Implements a fully responsive design grid for property listings and flexbox for layouts.
Conclusion
HomeHive is a clean, well-organized Vanilla HTML/CSS/JS application. Because it heavily relies on static dummy data (rentalListings) and local execution, there's no backend connection or database currently wired up. The codebase follows fundamental frontend best practices like DOM event listening, string templating, localStorage usage, and semantic UI structure
